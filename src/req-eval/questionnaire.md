# 短途旅行规划 · 需求采集问卷(树形分支)

> 用途:一份**离线、可自助填写**的树形问卷。用户根据每题的选择进入不同的后续问题,
> 最终汇总成一份结构化《旅行需求表》,作为后续行程规划与规划结果评估的输入。
> 本问卷**不依赖 agent 实时交流**,分支逻辑由问卷自身的跳转规则承载。

---

## 1. 设计原则

1. **分支按需展开**:只问与用户处境相关的题目。没带孩子就不问孩子,不过夜就不问住宿。
2. **硬约束必答、软偏好可跳过**:规划无法进行的关键字段(天数、同行人、预算)设为必答;
   偏好类设默认值,跳过即取默认。
3. **用档位/标签代替自由数值**:预算用档位、风格用多选标签,降低填写成本,也便于映射到规划约束。
4. **每个选项都映射到一条规划约束**:问卷的每个答案都能被下游规划工具消费(见第 5 节)。
5. **就近收口**:任意分支最多 2~3 层深度,避免问卷过长。

## 2. 模块总览与触发关系

| 模块 | 名称 | 触发条件 |
|------|------|----------|
| A | 行程基础 | 所有人(入口) |
| B | 儿童相关 | A2 勾选「带儿童」 |
| C | 老人相关 | A2 勾选「带老人」 |
| D | 住宿相关 | A1 选择「两天一夜」及以上 |
| E | 宠物相关 | A2 勾选「带宠物」 |
| F | 出行与交通 | 所有人 |
| G | 主题与偏好 | 所有人 |
| H | 特殊约束 | 所有人(可整体跳过) |

## 4. 题目明细

> 标注:**[必答]** 缺失则规划无法进行;**[可选]** 跳过取默认值;**[多选]** 可勾多个。
> 「→」表示该选项触发的跳转。

### 模块 A · 行程基础

- **A1. 这次出行计划几天?** [必答][单选]
  - ○ 半天(3~5 小时)
  - ○ 一天
  - ○ 两天一夜 → **触发模块 D**
  - ○ 三天及以上 → **触发模块 D**

- **A2. 和谁一起出行?** [必答][多选]
  - □ 独自一人
  - □ 情侣/伴侣
  - □ 朋友(成人结伴)
  - □ 带儿童 → **触发模块 B**
  - □ 带老人 → **触发模块 C**
  - □ 带宠物 → **触发模块 E**

- **A3. 人均预算档位?** [必答][单选]
  - ○ 经济(尽量省)
  - ○ 舒适(中等,体验优先)
  - ○ 不限(品质优先)

### 模块 B · 儿童(A2 含「带儿童」时出现)

- **B1. 孩子的年龄段?** [必答][多选]
  - □ 婴幼儿(0~3 岁) → **追加 B2**
  - □ 学龄前(3~6 岁) → **追加 B2**
  - □ 学龄(6~12 岁)
  - □ 青少年(12 岁以上)

- **B2. 是否需要以下设施?**(仅 0~6 岁出现)[可选][多选]
  - □ 婴儿车友好(平缓路面/电梯)
  - □ 母婴室/哺乳点
  - □ 儿童餐/餐椅

- **B3. 是否希望包含亲子/科普类活动?** [可选][单选]
  - ○ 是,优先安排  ○ 顺路即可  ○ 不需要(默认)

### 模块 C · 老人(A2 含「带老人」时出现)

- **C1. 老人的步行耐受度?** [必答][单选]
  - ○ 可长时间步行
  - ○ 适度步行,需间歇休息(默认)
  - ○ 需尽量少走/无障碍 → **追加 C2**

- **C2. 是否需要无障碍通行?**(仅上一题选「无障碍」出现)[可选][多选]
  - □ 轮椅可达  □ 必须有电梯/坡道  □ 避免长楼梯

- **C3. 是否需要避免高强度项目?** [可选][多选]
  - □ 避免爬山/爬高  □ 避免高海拔  □ 避免暴晒/严寒久站

### 模块 D · 住宿(A1 选「两天一夜」及以上时出现)

- **D1. 住宿档位?** [必答][单选]
  - ○ 经济(连锁/民宿)  ○ 舒适(中端酒店)  ○ 品质(高端/特色)

- **D2. 住宿位置偏好?** [可选][单选]
  - ○ 市中心(就近吃喝玩)
  - ○ 景区附近(省通勤)
  - ○ 交通枢纽附近(进出方便)
  - ○ 无所谓(默认,由规划决定)

- **D3. 房型/特殊需求?** [可选][多选]
  - □ 大床房  □ 双床房  □ 家庭房/亲子房  □ 含早餐  □ 可停车

### 模块 E · 宠物(A2 含「带宠物」时出现)

- **E1. 宠物类型与体型?** [必答][单选]
  - ○ 小型犬/猫  ○ 中大型犬  ○ 其他

- **E2. 需要哪些宠物友好条件?** [可选][多选]
  - □ 宠物友好住宿  □ 宠物可入餐厅/景区  □ 自驾(便于携带)

### 模块 F · 出行与交通(所有人)

- **F1. 倾向的出行方式?** [必答][单选]
  - ○ 自驾  ○ 公共交通  ○ 打车/包车  ○ 都可以(默认)

- **F2. 单程通勤最长可接受多久?** [可选][单选]
  - ○ ≤ 30 分钟  ○ ≤ 1 小时(默认)  ○ ≤ 2 小时  ○ 不介意

### 模块 G · 主题与偏好(所有人)

- **G1. 这次想要的体验?** [必答][多选]
  - □ 自然风光  □ 历史文化  □ 美食 → **追加 G2**
  - □ 购物  □ 网红打卡  □ 休闲放松
  - □ 户外运动 → **追加 G3**  □ 亲子

- **G2. 饮食禁忌/偏好?**(选了「美食」出现)[可选][多选]
  - □ 无(默认)  □ 忌辣  □ 清真  □ 素食  □ 海鲜过敏  □ 其他(填写:______)

- **G3. 户外强度偏好?**(选了「户外运动」出现)[可选][单选]
  - ○ 轻度(散步/骑行)  ○ 中度(徒步)  ○ 高强度(登山/越野)

- **G4. 行程节奏?** [必答][单选]
  - ○ 紧凑(多景点,效率优先)
  - ○ 适中(默认)
  - ○ 悠闲(少而精,体验优先)

### 模块 H · 特殊约束(可整体跳过)

- **H1. 天气/季节敏感?** [可选][多选]
  - □ 怕晒,尽量室内/遮阳  □ 怕冷,减少户外久站  □ 雨天需备选室内方案

- **H2. 有没有必去的地点?** [可选][自由填写]
  - 必去清单:________________(规划中作为硬约束优先满足)

- **H3. 其他禁忌或备注?** [可选][自由填写]

## 6. 选项 → 规划约束映射(供下游使用)

| 字段 | 约束类型 | 在规划中的作用 |
|------|----------|----------------|
| `duration` / `overnight` | 硬约束 | 决定天数框架、是否需要住宿模块 |
| `companions` | 硬约束 | 触发儿童/老人/宠物的可达性与适配过滤 |
| `budget_tier` | 硬约束 | 餐饮/住宿/门票档位筛选与总预算上限 |
| `kids.facilities` / `elderly.accessibility` | 硬过滤 | 剔除不满足无障碍/母婴设施的地点 |
| `elderly.avoid` / `weather_sensitivity` | 硬过滤 | 排除爬山、高海拔、暴晒久站等项目 |
| `accommodation.*` | 偏好+过滤 | 住宿选址与房型筛选 |
| `transport.mode` / `max_commute` | 硬约束 | 通勤方式与单程时长上限,影响选址半径 |
| `preferences.themes` | 软偏好(打分) | 景点/活动类型的相关性排序 |
| `preferences.diet` | 硬过滤 | 餐厅筛选 |
| `preferences.pace` | 软约束 | 每日景点数量与时间留白 |
| `must_visit` | 硬约束(最高优先) | 必须纳入行程,其余围绕它排布 |

---

## 7. 与规划结果评估的衔接

本需求表同时是**规划结果评估**的"对照基准":
- **约束满足度**:`硬约束/硬过滤` 字段逐项核对行程是否满足(可程序化判定)。
- **软偏好覆盖率**:`themes` / `pace` 等被行程体现的比例(LLM + rubric 判定)。
- **必去清单命中**:`must_visit` 是否全部纳入(确定性判定)。

> 即:问卷产出的每一条约束,都对应评估阶段的一个可检查项,形成"采集 → 规划 → 评估"的闭环。
